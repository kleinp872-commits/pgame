# 墨守 · Ink Guard — 视觉特效 / HUD / 音效技术升级设计

日期：2026-07-03
状态：已确认（用户批准）
目标文件：`mo-shou_3.html`（保持单文件、零依赖、IIFE 结构）

## 背景与目标

现有游戏玩法完整但表现层平淡：命中无反馈力度、墨晕特效单一、HUD 是静态
DOM 文本、无声音。本次升级**只改表现层与外围技术**，不改核心玩法数值与
判定逻辑。

成功标准：
- 每次划杀有明确的视听反馈（震屏、顿帧、音效、飞溅）。
- 整体水墨风格更浓（纸纹、飞白、残墨、化墨死亡）。
- HUD 信息更丰富（combo、飘字、最高分）且更具水墨美术感。
- 在移动端保持流畅（离屏缓存 + 对象池）。
- 仍是双击即玩的单 HTML 文件。

## 明确不做（YAGNI）

- 不加新敌人种类；不改 `LEVELS` 数值、生成逻辑、碰撞判定。
- 不做循环背景音乐（只做合成点缀音效）。
- 不拆分文件、不引入构建工具或外部资源。

## 总体结构

`<script>` IIFE 内部按注释分区新增/改造模块，顺序如下：

```
audio    — Web Audio 合成音效（sfx 对象 + 静音开关）
juice    — 屏幕震动、顿帧、震动反馈（vibrate）
fx       — 墨滴飞溅、化墨碎片、残墨层、闪电、转场、雾层
hud      — combo、飘字、印章生命、滚动分数、危险晕影
core     — 现有 update/render 挂接以上模块
persist  — localStorage 最高分
```

## 1. 打击感（juice）

- **屏幕震动** `shake(power)`：render 时对 `gctx` 整体 `translate` 随机偏移，
  指数衰减。命中 power≈3，斩杀 ≈6，敌人越线 ≈10。震动上限封顶，防叠加过猛。
- **顿帧 hit-stop**：斩杀瞬间设 `freezeT=50`（ms）；`update` 在 freeze 期间
  跳过游戏逻辑（特效层仍更新），制造"斩中"停滞感。仅斩杀触发，命中不触发。
- **方向性墨滴飞溅**：`splatter(x,y,dirAngle)` 替换均匀 `burst` —— 粒子速度
  锥形分布在划动方向 ±40°，大小不一、带重力下坠、落点留微小墨点。
- **Vibration API**：`buzz(ms)` 封装 `navigator.vibrate`（不支持则静默）。
  命中 10ms / 斩杀 25ms / 丢命 80ms。

## 2. 水墨渲染升级（fx）

- **宣纸纹理**：`resize()` 时在离屏 canvas 程序生成纸纤维噪点（随机短纤维
  线 + 低频斑块），以低透明度叠印到 `#bg`。只在 resize 重算。
- **笔触飞白拖尾**：`strokeTrail` 渲染改为三层 —— 中心浓墨线（宽随速度变
  化，快则细）、两侧 30% 透明淡墨、沿线随机跳过的短段（飞白）。收笔端半径
  收细。
- **残墨层**：新增第三个 canvas `#stain`（位于 bg 与 game 之间）。每次斩杀
  把一枚淡墨晕 `drawImage` 到该层；层整体每隔数秒做一次
  `globalAlpha` 递减重绘实现缓慢淡去（≈20 秒消失）。上限控制：淡出用
  节流（每 2s 一次 fade pass），不逐帧操作。
- **化墨死亡**：`killEnemy` 时按敌人半径生成 4–6 片墨块碎片（不规则多边形
  路径），各自带角速度、初速度（继承划动方向）、下坠与边缘晕开（半径微
  涨 + alpha 衰减），约 600ms 消散。原 `inkBloom` 保留作衬底。

## 3. 全屏氛围（fx）

- **雷击**：`resolveCast(true)` 时序列 —— ①全屏白闪（0.1s）→ ②程序绘制
  分叉闪电（主干折线 + 递归分支，发光描边，寿命 250ms）→ ③敌人按 y 序
  每 60ms 依次化墨（非同时清屏）。
- **危险晕影**：HP<30% 时 render 尾部叠加边缘径向渐变朱砂晕影，
  呼吸式 alpha（sin 波动），HP 越低越浓。
- **关卡转场**：`beginLevelClear` 触发墨浪 —— 一条带不规则前缘的浓墨横带
  从左扫到右（≈700ms），扫过处短暂全墨，浪尾露出下一关标题
  「第 N 關」大字毛笔字号淡入 → 淡出后 `startLevel`。转场期间不生成敌人
  （沿用现有 `levelTransition` 机制，时长调整为匹配动画）。
- **背景薄雾**：2–3 条低透明度横向雾带（离屏预渲染模糊椭圆），速度极慢。
  因雾需要动而 bg 层是静态的，统一在 **game 层每帧最先绘制**（位于所有游戏
  元素之下），每帧仅做 `drawImage` 平移，开销可忽略。

## 4. HUD 重做（hud）

- **印章生命**：`.dot` 改为小型朱砂印章方块（CSS 绘制「守」字样式即可）。
  丢命时该印章播放碎裂动画（CSS：裂缝伪元素 + 抖动 + 淡出），保留灰印占位。
- **滚动分数**：分数变化时数字快速递增滚动（js 补间，≈300ms），并做
  scale 1→1.15→1 回弹。
- **飘字**：击杀点冒 `+N` 墨色小字，上飘 + 淡出（canvas 绘制，走对象池）。
- **连击 combo**：斩杀后 2s 内再斩杀则 `combo++`，HUD 中央偏上显示
  `連斬 ×N`（毛笔字风格，出现时放大回弹）。combo≥2 起每级额外
  +2 分/杀（`score += ptsPerHit + (combo-1)*2`，上限 +20）。超时归零并淡出。
- **HP 条动效**：受击时 HP 条红闪 2 次 + 水平抖动（CSS animation，js 加
  class 触发）；波次进度条填充改为墨色线性渐变 + 缓慢流动
  （background-position 动画）。
- **新敌人登场提示**：`startLevel(n)` 时对比上一关 animals，若出现新类型，
  屏幕中央显示 1.5s 名牌：「墨蝌 · 一斬」「墨蛸 · 二斬」「墨蜇 · 二斬」
  （hits 取该关配置）。
- **布局微调**：统一四角 padding 至 16px；标题字号 22→18；按钮组与生命
  行间距收紧。不移动元素归属的角落。

## 5. 音效（audio，Web Audio 程序合成）

单例 `AudioContext`（首次用户交互时创建/resume）。`sfx` 对象暴露：

| 音效 | 触发 | 合成配方（概述） |
|---|---|---|
| `slash` | 命中敌人 | 白噪声 80ms + 带通(2–4kHz) 扫频下滑 + 快速衰减包络 |
| `kill` | 斩杀 | slash 基础上叠低频正弦 120Hz 短促"扑"声 |
| `thunder` | 雷击成功 | 棕噪声 1.2s 低通渐闭 + 50Hz 正弦轰底 + 起始咔裂(高通噪声瞬态) |
| `hurt` | 敌人越线 | 三角波 220→110Hz 下滑 300ms，如闷钟 |
| `clear` | 过关 | 五声音阶(宫商角徵羽)上行琶音，正弦+轻微延音，似古琴泛音 |
| `chime` | 开始/结束界面 | 正弦 880Hz + 谐波，长衰减，如磬一声 |
| `combo` | combo≥3 时每次斩杀 | 短正弦 pitch 随 combo 递升 |

- 右上角新增静音按钮（🔇/🔊 风格化为「音」字按钮），状态存 localStorage。
- 所有节点即用即建，接主 `GainNode`（总音量 + 静音控制）。

## 6. 性能（core）

- **离屏缓存**：纸纹、雾带、山体背景均预渲染离屏 canvas，运行时仅
  `drawImage`。
- **对象池**：粒子（飞溅）、飘字、墨碎片使用固定容量池（如 256/32/64），
  `alive` 标记复用，`update` 不再 `filter()` 重建数组（enemies 数量小，
  保持现状不动）。
- **帧内零分配原则**：热路径避免每帧创建对象/闭包/渐变；渐变仅在必要特效
  处按需创建（inkBloom 保持现状——数量少可接受）。

## 7. 最高分持久化（persist）

- key：`inkguard_best = {score, level}`（JSON）。
- `gameOver()` 时对比并写入；结束界面显示「本局 SCORE X · 最佳 Y」，
  破纪录时替换为「新紀錄！」+ 印章红字样式。
- localStorage 不可用（隐私模式）时 try/catch 静默降级。

## 错误处理

- Web Audio / Vibration / localStorage 全部特性检测 + try/catch，缺失时
  静默降级，游戏功能不受影响。
- 残墨层与对象池有硬上限，长时间游玩不累积内存。

## 测试方案

无测试框架（单文件游戏）。验证方式：
1. 浏览器手动冒烟：桌面 Chrome + 移动端模拟（触摸事件、DPR=2/3）。
2. 每个特效的触发路径逐一人工核对（命中/斩杀/越线/雷击/过关/游戏结束）。
3. 性能：DevTools Performance 录制 1 分钟高强度游玩，帧率不低于 55fps
  （桌面）；检查无持续增长的堆内存。
4. 降级：Console 禁用 localStorage / 无声环境验证不报错。

## 实现顺序（供计划参考）

1. juice（震屏/顿帧/飞溅/vibrate）— 最大手感收益
2. audio（sfx + 静音钮）
3. fx-水墨（纸纹/飞白/残墨/化墨）
4. fx-氛围（闪电/晕影/转场/雾）
5. hud（印章/滚动分/飘字/combo/名牌/条动效）
6. persist + 性能收尾（对象池化、离屏缓存核查）
